# Purpose

Covenant RPC is meant to simply and elegantly solve a couple of problems:
- I want to do things on the server from the client and get consistent responses
- I want my app to be able to automatically refresh outdated info
- I want to have a simple, scalable, realtime channel system

Its non-functional goals are to:
- Be simple for AI agents (and therefore, humans as well) to use and understand
- Move as many errors as possible to the type layer
